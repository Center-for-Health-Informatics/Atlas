# AMD ‚Üí ESM Migration Status

There's no single doc that's tracked this as it happened ‚Äî the real history lives across git log messages, code comments, and (until now) a Claude memory file nobody else could see. This is an attempt to consolidate it as of 2026-07-07. Verify against current code/git log before trusting specifics ‚Äî this will drift.

## Where things stand

**Source conversion: essentially done.** Of ~785 JS files under `js/`, only 9 non-build-output files still use `define(...)` ‚Äî and all 9 are third-party/vendor code, not app source:
- RequireJS plugin shims: `js/extensions/plugins/{text,css.min,less,optional}.js`
- Vendor UMD libs: `jquery.ddslick.js`, `d3.slider.js`, `d3-scale-chromatic.1.3.0.min.js`, `jnj.chart.js`
- `js/extensions/bindings/knockout.selectOnFocus.js`

Git history for this: `3ab53419` (phases 0 & 1), `a7cd1b94` (phase 2: batch-converted 620+ files), `5c76a6a9` ("mid AMD‚ÜíESM migration"), plus assorted individual bug fixes since.

**Dev server: works.** `npm run dev` runs Vite at whatever host/port is configured (see `vite.config.mjs`), proxying `/WebAPI` to the webapi container. App boots, renders nav, and ‚Äî as of 2026-07-08 ‚Äî Home, Vocabulary Search, Concept Set creation/editing, Cohort Pathways, Cohort Definitions (create/add-criteria/save), and the "New" editors for Estimation, Prediction, and Characterizations have all been smoke-tested end-to-end with a real browser (Playwright) against the `https://dev.lastchance.pub/webapi/` test instance, with bugs found and fixed along the way (see `CHANGELOG.md` ‚Äî includes a big one: ~53 cohort-builder/viewer components had no view model wired up at all). Workflows for all of these are now documented in `USAGE.md`. Not yet smoke-tested: actually running Generation/Execution against a data source (needs ACHILLES run against the Synthea CDM first), Cohort Pathways generation, full Reusables flow. Data Sources dashboards are expected to be empty until ACHILLES has been run against the Synthea CDM.

**Production build: fixed as of 2026-07-08.** `build:dev` now runs `vite build --config vite.config.mjs` directly; `build` and `build:docker` compose on top of it unchanged. `build/optimize.js` (the old RequireJS optimizer + Babel + Terser path) and `build/polyfill.js` (only used by it, superseded by `@vitejs/plugin-legacy`) are deleted. The `compress` script (a separate `terser` pass) is gone too ‚Äî `vite build` already minifies. Verified: `npm run build`, `npm run build:docker`, and `npm test` all pass; the resulting `js/assets/bundle/` is a full chunked build (1264 modules transformed, `index.html` references real hashed asset chunks) and was smoke-checked via `vite preview`.
  - This also exposed and fixed a latent bug in `genversion`: the old `genversion -s js/version/version.js && r_js -convert js/version js/` pipeline produced an AMD `define()`-wrapped `js/version.js`, which the ESM app can't import (`"version" is not exported by "js/version.js"`) ‚Äî this had been masked until now because `build/optimize.js` silently no-op'd before anything tried to actually load the bundle. Replaced with a single step: `genversion -e -s js/version.js`, which writes plain ESM (`export const version = ...`) directly ‚Äî no intermediate directory, no `r_js -convert`, no `rimraf` needed.
  - `requirejs` was only ever used by `build/optimize.js` and by `r_js -convert` in the old `genversion` step ‚Äî both gone now, so the `requirejs`, `@babel/*`, `terser`, and `html-document` devDependencies were dropped from `package.json` (not deferred to step 6 as originally guessed; turned out nothing else needed them).

**Tests: fixed ‚Äî now on Node's built-in test runner, 18/18 passing.** Jest is gone (`jest.config.js` deleted, `jest` dropped from `package.json`). It had depended on a RequireJS shim (`requireAmd()`/`requirejsInstance`) to load AMD-style modules, which stopped working once app source moved to ESM (every test failed with `SyntaxError: Unexpected token 'export'`). `npm test` now runs `node --import ./tests/register-hooks.mjs --test`, and all 5 existing test files (`AutoBind`, `BemHelper`, `CommonUtils`, `DataTypeConverterUtils`, `ExceptionUtils`) were ported to plain ESM imports + `node:assert/strict`. `package.json` now declares `"type": "module"` (verified this doesn't break `vite build` or `vite --config`). The one module-resolution wrinkle ‚Äî `CommonUtils.js` importing Vite-only bare aliases (`appConfig`, `atlas-state`, etc.) that plain Node can't resolve ‚Äî is handled by a small custom resolve hook (`tests/hooks.mjs`, registered via `tests/register-hooks.mjs`) that redirects those specifiers to stub files in `tests/stubs/`. See `CHANGELOG.md` (2026-07-07) for details.

**Docs: `atlas/CLAUDE.md` is stale.** It was last edited at the "initial reformatting to neostandard" commit (`5b3d0360`), which is *before* all the AMD‚ÜíESM migration commits. It currently describes the codebase as pure AMD/RequireJS with "no dev server" ‚Äî both no longer true. Worth updating so future sessions (Claude or otherwise) aren't working from an inaccurate architecture description.

**Lint:** `npm run lint` runs (doesn't crash) and, as of 2026-07-08, reports ~3700 problems (down from ~4100) ‚Äî the large majority auto-fixable stylistic issues from the neostandard reformat. The `no-undef` category (248 errors) has been fully cleared: those were all real migration bugs (missing `$`/`_`/`saveAs`/`Prism`/`moment` imports left over from RequireJS's global shims, plus several implicit-global-assignment bugs that only worked in non-strict AMD scripts) ‚Äî see `CHANGELOG.md` (2026-07-08) for the full list, which includes some genuine correctness bugs (e.g. `AuthAPI.isPermitted()` was silently broken for every permission check). Remaining lint problems are stylistic (`@stylistic/*`, `eqeqeq`, `no-unused-vars`, `prefer-const`) ‚Äî not yet cleaned up post-reformat.

**Debug scaffolding still in place, meant to be temporary:**
- `js/main.js` ‚Äî sequential `log()` calls that write each import step to `console.log` and `document.title`, meant to help pinpoint which import broke during the migration.
- `index.html:5` ‚Äî a global `window.addEventListener('error', ...)` handler that also writes to `document.title`, same purpose.

Both were left in deliberately to make module-load failures visible during the migration. Worth removing once things feel stable enough not to need them ‚Äî they're currently still present.

**`vite.config.mjs` ‚Äî "Phase 3" referenced but not defined anywhere else.** A comment there reads `// AMD plugin aliases (kept for transition; removed after Phase 3)`, aliasing `text!`, `less!`, `css!` (RequireJS plugin prefixes) to their Vite equivalents. No other doc defines what "Phase 3" consists of ‚Äî reasonable guess, given everything else above, is that it's: (1) switch the build scripts to `vite build`, (2) drop the old RequireJS-based build/test tooling, (3) drop these transitional aliases once nothing references the old `text!`/`less!`/`css!` prefixes, (4) remove the debug scaffolding above.
